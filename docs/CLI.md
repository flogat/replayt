# CLI reference

All commands support Typer's `--help`. Most read/write commands accept `--log-dir`, optional `--log-subdir`, and optional `--sqlite` unless noted; defaults match [`CONFIG.md`](CONFIG.md) when you use project config (and `REPLAYT_LOG_DIR` when the CLI default log root applies). On **Unix**, new per-run **`*.jsonl`** files default to **owner-only (`0600`)** permissions unless you set **`REPLAYT_JSONL_POSIX_MODE`** (see [`CONFIG.md`](CONFIG.md)).

When the **`replayt`** entry point is not on your **`PATH`** (minimal containers, some **`pip install --user`** shells), run the same CLI as **`python -m replayt`** with the same subcommands and flags, for example **`python -m replayt version --format json`**.

## `replayt init [--path DIR] [--force] [--ci github]`

Write `workflow.py` or `workflow.yaml`, `inputs.example.json`, `.env.example`, a local `.replaytrc.toml` (`target` + `inputs_file` for this directory), and a `.gitignore` snippet (`.replayt/`, `.env`, ...). Refuses to overwrite unless `--force`. **`--template`** picks a scaffold (`basic`, `approval`, `tool-using`, `yaml`, `issue-triage`, `publishing-preflight`); **`--list`** prints every key with a one-line summary, a copy-paste **`replayt init --template KEY`** line, and a portable **`replayt run <workflow> --inputs-file …`** line that matches that template’s filenames (same ergonomics as **`replayt try --list`**). **`--list --output json`** emits **`replayt.init_templates.v1`** with per-template **`workflow_file`**, **`inputs_file`**, **`llm_backed`**, **`summary`**, and a **`cli`** object: **`init_here`**, **`init_with_ci_github`**, **`doctor_target`**, explicit-path **`validate_explicit`**, **`dry_check_explicit`**, **`run_explicit`**, **`ci_dry_check_explicit`**, project-default **`dry_check`** / **`run`**, and LLM templates add **`dry_run`**. Cannot combine **`--list`** with **`--ci`**. **`--ci github`** also writes `.github/workflows/replayt.yml` (replace `CHANGE_ME_MODULE:wf` with your target). The command now prints copy-paste next steps with shell-specific venv activation lines, **`replayt doctor --skip-connectivity --target ...`**, then bare **`replayt run --dry-check`** / **`replayt run`** commands that use the generated project config; LLM-backed templates start with a suggested **`--dry-run`** before the live run.

## `replayt init-gitignore [--path DIR]`

Append the same replayt-focused lines **`replayt init`** would add (`.replayt/`, `.env`, `.venv/`, `__pycache__/`) to **`.gitignore`** under **`--path`** (default **`.`**). Creates **`.gitignore`** when it is missing; otherwise appends only lines that are not already present. Use this when you already have a workflow (for example from **`replayt try --copy-to`**) and only need the ignore snippet.

## `replayt init-env-example [--path DIR] [--force]`

Write the same **`.env.example`** template **`replayt init`** generates (commented **`OPENAI_API_KEY`**, optional **`OPENAI_BASE_URL`** / **`REPLAYT_*`** hints, including commented **`REPLAYT_TARGET`** and **`REPLAYT_INPUTS_FILE`** pointers to [`CONFIG.md`](CONFIG.md)). Refuses to overwrite an existing file unless **`--force`**. Use this when you already have **`workflow.py`** (or **`MODULE:VAR`**) but teammates still need a copy-paste env template without re-running a full scaffold.

## `replayt run [TARGET]`

Run a workflow from a module reference, Python file, or YAML file. **`TARGET` is optional** when env **`REPLAYT_TARGET`** is set or project config defines **`target`** (see [`CONFIG.md`](CONFIG.md)); an explicit **`TARGET` argument always wins**. With no resolved default, Typer fails fast and prints copy-paste examples (**`replayt run replayt_examples.e01_hello_world:wf …`**, **`replayt try --list`**, **`replayt init`**) plus doc pointers so a first invocation is not a dead end. Common flags: `--output text|json`, `--log-mode ...`, `--redact-key FIELD` (repeatable, scrubs matching structured keys from logged payloads), `--resume`, `--tag key=value` (repeatable), `--metadata-json '{...}'` (`run_started.run_metadata`), **`--experiment-json '{...}'`** (`run_started.experiment`, merged into per-call LLM `effective`), **`--policy-hook-context-json '{...}'`** or **`@path`** / **`@-`** (hook-only JSON as **`REPLAYT_POLICY_HOOK_CONTEXT_JSON`** on policy subprocesses; not written to JSONL; overrides env **`REPLAYT_POLICY_HOOK_CONTEXT_JSON`** and project **`policy_hook_context_json`**; see [`CONFIG.md`](CONFIG.md)), `--log-subdir NAME` (one segment under the resolved log root), `--timeout SECONDS`, **`--inputs-json`** or **`--inputs-file PATH`** (mutually exclusive; **`--inputs-json @path.json`** reads a file, **`--inputs-json @-`** reads a UTF-8 JSON object from stdin; **`--inputs-file -`** is the same stdin shortcut; when both flags are omitted, defaults may come from env **`REPLAYT_INPUTS_FILE`** (including the literal value **`-`** for stdin) or project **`inputs_file`** (see [`CONFIG.md`](CONFIG.md))), repeatable **`--input key=value`** overrides for the common case where a whole JSON blob is overkill, `--dry-run` (placeholder LLM), `--dry-check` (validate graph + optional JSON blobs; with **`--output json`** prints a `replayt.validate_report.v1` object and exits `1` if not ok), `--strict-graph` (fail validation when there are 2+ states but no declared transitions). Graph validation runs before every real execution (not only `--dry-check`). `run_started` now also records a safe `runtime` snapshot (engine/store class, workflow contract fingerprint, plus non-secret LLM settings such as base URL and model). **`--output json`** emits **`replayt.run_result.v1`**.

**`--input` details:** each flag is `key=value`. Dotted keys build nested objects (`--input issue.title=Crash --input issue.body="Stacktrace..."`). Values are parsed as JSON when possible, so `2`, `false`, `null`, `["a"]`, and `{"k":"v"}` become structured values; anything else stays a string. You can combine **`--input`** with **`--inputs-json`** or **`--inputs-file`**; later `--input` paths override the same keys from the base object.

**Missing inputs file:** when **`--inputs-file`**, **`--inputs-json @path`**, **`REPLAYT_INPUTS_FILE`**, or **`[tool.replayt] inputs_file`** points at a path that does not exist, the error explains which source was used (explicit flag vs env vs project config) so you can fix the path, run **`replayt config`** to see resolved defaults, or override with **`--inputs-file`** / **`--inputs-json`** for that invocation.

If project config defines **`run_hook`** (argv list) or env **`REPLAYT_RUN_HOOK`** is set, that command runs before execution with `REPLAYT_TARGET`, `REPLAYT_RUN_ID`, `REPLAYT_RUN_MODE` (`run` or `resume`), `REPLAYT_LOG_DIR`, `REPLAYT_RUN_JSONL` (resolved path to `<run_id>.jsonl` under that log root; the file may not exist yet on a brand-new run), `REPLAYT_LOG_MODE`, `REPLAYT_FORBID_LOG_MODE_FULL`, `REPLAYT_REDACT_KEYS_JSON` (sorted JSON array of configured redact field names), `REPLAYT_REPLAYT_VERSION` (installed **replayt** version string), `REPLAYT_DRY_RUN`, optional `REPLAYT_SQLITE`, `REPLAYT_WORKFLOW_CONTRACT_SHA256`, `REPLAYT_WORKFLOW_NAME`, and `REPLAYT_WORKFLOW_VERSION` (from the loaded workflow contract; same digest as on **`run_started`**), optional `REPLAYT_WORKFLOW_ENTRY_PATH` (absolute path to the workflow `.py` / YAML file or the imported module's `__file__` when replayt can resolve it), optional `REPLAYT_RUN_STARTED_TS` (ISO 8601 UTC `ts` from the first **`run_started`** line when **`--resume`** is used and that envelope field is non-empty; omitted on brand-new runs before **`run_started`** exists), optional `REPLAYT_RUN_STARTED_RUNTIME_JSON` (canonical sorted JSON of that line's **`runtime`** object; `{}` when **`runtime`** is absent or null; omitted on brand-new runs the same way as **`REPLAYT_RUN_STARTED_TS`**), and normalized JSON object strings in `REPLAYT_RUN_INPUTS_JSON`, `REPLAYT_RUN_TAGS_JSON`, `REPLAYT_RUN_METADATA_JSON`, and `REPLAYT_RUN_EXPERIMENT_JSON` when those values are present on the CLI, plus optional `REPLAYT_WORKFLOW_META_JSON` when `Workflow.meta` (excluding `llm_defaults`) is non-empty (the same object emitted as `run_started.workflow_meta`), and optional `REPLAYT_POLICY_HOOK_CONTEXT_JSON` when **`--policy-hook-context-json`**, env **`REPLAYT_POLICY_HOOK_CONTEXT_JSON`**, or project **`policy_hook_context_json`** supplies hook-only JSON (not logged on **`run_started`**). Policy wrappers can use that single env block to check change tickets, deployment tiers, tag-based allowlists, or approved workflow digests before replayt writes new events. Successful runs record a compact breadcrumb under **`run_started.runtime.policy_hooks.run`** (`source`, `argv0`, `arg_count`) so the log shows that trusted external code was part of the gate. Non-zero exit aborts before replayt writes new events. A default 120s wall-clock limit applies unless you set **`run_hook_timeout`** / **`REPLAYT_RUN_HOOK_TIMEOUT`** (`<= 0` = no limit). See [`CONFIG.md`](CONFIG.md).

**Exit codes:** `0` completed, `1` failed, `2` paused (approval required).

Listing, inspection, export, and seal helpers use **`1`** for missing runs, invalid arguments, or verification failures so **`2`** stays reserved for approval pauses on **`replayt run`** / **`replayt ci`**.

**CI-style artifacts (optional):** set env **`REPLAYT_JUNIT_XML`** to a path to write the same minimal JUnit XML as **`replayt ci --junit-xml`**, **`REPLAYT_SUMMARY_JSON`** to write the same machine-readable JSON artifact as **`replayt ci --summary-json`**, and **`REPLAYT_GITHUB_SUMMARY=1`** to append the same markdown summary as **`replayt ci --github-summary`** when **`GITHUB_STEP_SUMMARY`** (GitHub Actions) or **`REPLAYT_STEP_SUMMARY`** (any CI: explicit path to append) is set; when both are set, **`GITHUB_STEP_SUMMARY`** wins. When a summary path is in effect (flag or **`REPLAYT_SUMMARY_JSON`**), the written **`replayt.ci_run_summary.v1`** includes **`cwd`** (resolved process working directory when the summary is written), **`replayt_version`**, **`python_version`**, **`python_implementation`** (for example **CPython** vs **PyPy**), **`python_cache_tag`** (from **`sys.implementation.cache_tag`** when present, else **`null`**; same ABI hint as **`replayt version --format json`** → **`python.implementation.cache_tag`**), **`python_executable`**, **`platform`**, **`machine`** (CPU architecture from **`platform.machine()`**), **`filesystem_encoding`** (**`sys.getfilesystemencoding()`**), **`stdout_encoding`** (from **`sys.stdout.encoding`** when present), **`ci_marker_env`** (booleans only: which common CI vendor env vars are set; **values are never copied**), **`host_clock_utc_offset_minutes`** (local wall clock offset from UTC when known, else **`null`**), and on POSIX **`ulimit_nofile_soft`** / **`ulimit_nofile_hard`** from **`resource.getrlimit(RLIMIT_NOFILE)`** when the call succeeds (otherwise **`null`**), and optional env **`REPLAYT_CI_METADATA_JSON`** may hold a JSON **object** of pipeline fields (build id, commit, job URL); valid values are merged into the summary payload as **`ci_metadata`** (invalid JSON or a non-object fails fast before the run starts). **`replayt config --format json`** and **`replayt doctor --format json`** preview those env-driven artifact sinks so CI can catch missing parents or a missing **`GITHUB_STEP_SUMMARY`** export before the real run. See [`CONFIG.md`](CONFIG.md) and [`RECIPES.md`](RECIPES.md).

## `replayt try`

Run one of the packaged tutorial workflows without creating a local file first. Use **`--list`** to see the curated examples (each entry ends with a copy-paste **`replayt try --example KEY`** line), **`--example KEY`** to pick one, and optional **`--inputs-json`** / **`--inputs-file`** / repeatable **`--input key=value`** to override the sample payload. **`--list --output json`** emits **`replayt.try_examples.v1`** with a per-example **`cli`** object (`try_offline`, **`try_live`**, **`try_dry_check`**, **`copy_to_dot`**) so scripts and READMEs can surface the same strings without hard-coding keys. **`--print-snippet KEY`** prints **one** copy-paste shell line for the selected **`--example`** and exits (**no run**): **`try_offline`**, **`try_live`**, **`try_dry_check`**, **`copy_to_dot`**, **`target`** (only the **`MODULE:VAR`** string), **`run`** (a **`replayt run … --inputs-json …`** line with compact sorted JSON and POSIX-style quoting from **`shlex.quote`**), or **`run_dry_check`** (same plus **`--dry-check`**). For **`run`** / **`run_dry_check`**, input flags match the real try path (**`--inputs-json`**, **`--inputs-file`**, **`--input`**, and **`--customer-name`** for **`hello-world`**). **`--output json`** with **`--print-snippet`** emits **`replayt.try_print_snippet.v1`** (`example`, **`snippet_key`**, **`command`**). Cannot combine **`--print-snippet`** with **`--list`**, **`--copy-to`**, **`--live`**, **`--dry-check`**, **`--run-id`**, **`--timeout`**, **`--tag`**, or **`--redact-key`**. **Default:** placeholder LLM (`--dry-run`); pass **`--live`** to call the provider (needs `OPENAI_API_KEY`). `--customer-name` still customizes the default `hello-world` payload, and **`--input`** layers on top of that default example object rather than replacing it wholesale.

**Materialize locally:** **`--copy-to DIR`** copies the example module as **`workflow.py`**, **`inputs.example.json`**, and a local `.replaytrc.toml` wired to those files (no run). Refuses to overwrite unless **`--force`**. **`--output json`** prints **`replayt.try_copy.v1`** with absolute paths, including `project_config`. Text output now includes copy-paste follow-up commands (`doctor --skip-connectivity --target`, bare `run --dry-check`, and **`--dry-run`** first when the example is LLM-backed). Cannot be combined with **`--list`**, **`--live`**, **`--dry-check`**, **`--inputs-json`**, **`--inputs-file`**, **`--input`**, **`--run-id`**, or **`--timeout`**.

## `replayt ci [TARGET]`

Same behavior and flags as `replayt run` (including optional **`TARGET`** resolution via **`REPLAYT_TARGET`** / project **`target`**, **`--inputs-file`**, env/project default inputs, repeatable **`--input`**, **`--metadata-json`**, **`--experiment-json`**, **`--policy-hook-context-json`**), plus **`--strict-graph`**, **`--junit-xml PATH`** (minimal JUnit file for the run outcome), **`--summary-json PATH`** (machine-readable `replayt.ci_run_summary.v1`: workflow, `run_id`, status, final state, error, **`exit_code`** matching the CLI, **`target`**, **`cwd`**, resolved **`log_dir`**, optional **`sqlite`**, **`dry_run`**, wall-clock **`duration_ms`**, **`replayt_version`**, **`python_version`**, **`python_implementation`**, **`python_cache_tag`**, **`python_executable`**, **`platform`**, **`machine`**, **`filesystem_encoding`**, **`stdout_encoding`**, **`ci_marker_env`**, **`host_clock_utc_offset_minutes`**, POSIX **`ulimit_nofile_soft`** / **`ulimit_nofile_hard`** (or **`null`** off POSIX / when **`resource.getrlimit`** fails), and optional **`ci_metadata`** when **`REPLAYT_CI_METADATA_JSON`** is set), and **`--github-summary`** (append markdown to **`GITHUB_STEP_SUMMARY`** or **`REPLAYT_STEP_SUMMARY`** when set; GitHub wins if both are set). Prints a one-line reminder of exit codes for pipelines. See [`RECIPES.md`](RECIPES.md) for GitHub Actions examples.

`TARGET` can be:

- `module:variable` (e.g. `replayt_examples.e01_hello_world:wf`)
- `workflow.py` (must export `wf` / `workflow`, or exactly one top-level `Workflow` object)
- `workflow.yaml` / `workflow.yml` (requires `pip install replayt[yaml]`)

If `module:variable` fails with an import error, replayt prints hints: install your project editable (`pip install -e .`), add your source tree to `PYTHONPATH` when you use a `src/` layout without packaging, pass a `workflow.py` / YAML path instead, or run `replayt doctor --target TARGET` once imports succeed. When the module loads but a dependency import fails, the error names the missing module. When Python finds the package but import still fails (for example a circular import or `ImportError` raised at import time), replayt surfaces the underlying message and suggests `python -c "import MODULE"` plus `replayt doctor --skip-connectivity --target …` after you fix the graph. When the package file has a syntax error, replayt reports the parser location and suggests `python -m py_compile` on that file. Python file targets now get similar onboarding help: missing imports point at the missing package and suggest `pip install -e .` / `MODULE:VAR`, while syntax errors report the parser line before replayt exits. When the module loads but `:variable` is not a `Workflow` instance, replayt names the Python type (for example you pointed at a function, class, or plain value) and suggests `replayt doctor --skip-connectivity --target …` after you export `wf` / `workflow` correctly. Empty `module` or `variable` parts, or more than one `:` splitting the pair (for example `pkg.mod:wf:extra`), fail fast with a copy-paste example. When the attribute is missing but the module exports other top-level `Workflow` instances, replayt lists those names and suggests `replayt run` / `replayt doctor --skip-connectivity` using `wf` when that name exists (otherwise another exported name). YAML paths without PyYAML show the same `pip install replayt[yaml]` line as a CLI error with a retry hint instead of a traceback. When the target string is not a `module:variable` pair and the path does not exist, errors call out common mistakes: a dotted name without a colon (suggest `module:variable`), a basename missing `.py` / `.yaml` when that file exists beside you, passing a directory instead of a workflow file, or a non-workflow file extension. When the path **does** exist but is not a `.py` / `.yaml` / `.yml` workflow (often `inputs.json` passed as the first positional by mistake), errors explain that inputs use `--inputs-file` / project `inputs_file` and that `TARGET` must still be the workflow. On Windows, absolute paths such as `C:\\...\\workflow.py` are still treated as paths (the drive-letter colon is not read as `MODULE:VAR`).

## `replayt inspect RUN_ID`

Summary and event list for a run. **`--print-inputs`** prints only the **`run_started`** payload **`inputs`** object as **compact JSON** on stdout (same **`--log-dir`**, **`--log-subdir`**, and **`--sqlite`** as a normal read) so you can reuse the exact payload with **`replayt run TARGET --inputs-json …`** or **`--inputs-json @path`**. Omit **`--output`**, **`--json`**, and timeline filters when using **`--print-inputs`**; values may include **`{"_redacted": true}`** placeholders when the original run used **`--redact-key`** / **`redact_keys`**. **`--output markdown`** prints a short **paste-friendly** blurb (workflow, status, contract digest when present, last event time, and the same attention line **`replayt runs`** uses for paused/failed runs) plus suggested next-step commands. **`--style stakeholder`** (default) uses the usual **`## replayt run`** heading (with the run id) and copy-paste lines for **`replayt report` / `replayt replay` / `replayt bundle-export`** with **`--style stakeholder`** and **`{run_id}-stakeholder-bundle.tar.gz`**. **`--style support`** uses a **`## Support handoff`** heading that includes the run id (parallel to the stakeholder **`## replayt run`** heading) and copy-paste lines for the same three commands with **`--style support`** / **`--report-style support`** and **`{run_id}-support-bundle.tar.gz`**, aligned with support-oriented reports and archives. **`replayt resume …`** hints when the run is paused apply to both styles. **`--style support`** without **`--output markdown`** is an error; omit **`--style`** (or keep **`stakeholder`**) with **`--print-inputs`**. Markdown always summarizes the **full** run; combine **`--event-type`**, **`--note-kind`**, **`--finish-reason`**, **`--tool`**, **`--structured-schema`**, or **`--llm-model`** with **`--output markdown`** is an error. **`--output json`** (or legacy **`--json`**) emits **`replayt.inspect_report.v1`** with **`run_id`**, **`summary`**, and **`events`**. When present in the log, **`summary.workflow_contract_sha256`** is the workflow contract fingerprint from `run_started.runtime.workflow.contract_sha256`, and text output prints the same digest on its own line for quick upgrade triage. Repeat **`--event-type TYPE`** to restrict the printed / JSON `events` array to matching JSONL `type` values (OR semantics); **`summary`** still describes the full run. Repeat **`--note-kind KIND`** to restrict the event list to matching **`step_note`** payload **`kind`** values (exact match; OR semantics). Without **`--event-type`**, **`--note-kind`** shows only matching notes. Repeat **`--finish-reason REASON`** to restrict the event list to matching **`llm_response`** payload **`finish_reason`** strings (exact match; OR semantics). Without **`--event-type`**, **`--finish-reason`** shows only matching responses. Repeat **`--tool NAME`** to restrict the event list to matching **`tool_call`** payload **`name`** strings (exact match; OR semantics). Without **`--event-type`**, **`--tool`** shows only matching tool calls. Repeat **`--structured-schema NAME`** to restrict the event list to matching **`structured_output`** / **`structured_output_failed`** payload **`schema_name`** strings, or tagged **`llm_request`** / **`llm_response`** lines with the same field (exact match; OR semantics), using the same rules as **`replayt runs`** and **`replayt stats`**. Without **`--event-type`**, **`--structured-schema`** shows only those matching events. Repeat **`--llm-model MODEL`** to restrict the event list to matching **`llm_request`**, **`llm_response`**, **`structured_output`**, or **`structured_output_failed`** lines whose logged model id matches (uses payload **`effective.model`** when present, else top-level **`model`**; exact match; OR semantics), using the same rules as **`replayt runs`** and **`replayt stats`**. Without **`--event-type`**, **`--llm-model`** shows only those LLM / structured lines. When filtering is active, JSON also includes **`event_type_filter`**, **`note_kind_filter`**, **`finish_reason_filter`**, **`tool_name_filter`**, **`structured_schema_filter`**, and/or **`llm_model_filter`**.

If there is no JSONL timeline for that id, replayt exits **1** (lookup / user error; not the same as exit **2** for paused runs) and prints a hint to run **`replayt runs --limit 10`**, repeating **`--log-dir`**, **`--log-subdir`**, and **`--sqlite`** only when you passed them on the failing command so the listing hits the same store, plus a short reminder to match the log store (**`--log-dir`** / **`REPLAYT_LOG_DIR`** / project **`log_dir`**, **`--log-subdir`**, **`--sqlite`**) you used when the run was created. The same hint appears for **`replay`**, **`report`**, **`report-diff`**, **`export-run`**, and **`bundle-export`** when the id is missing or empty. If **`RUN_ID`** fails validation (paths, slashes, or stray quotes), the error repeats the allowed character set and points at **`replayt runs`** and the **`run_id=`** line from **`replayt run`**.

## `replayt replay RUN_ID`

Recorded execution timeline **without** calling model APIs. **`--format html`** emits a self-contained page with inline CSS; **`--out PATH`** writes a file. **`--style`** matches **`replayt report`**: **`default`** prints every JSONL row, **`stakeholder`** or **`support`** drops **`llm_request`**, **`llm_response`**, **`tool_call`**, and **`tool_result`** lines in both text and HTML (HTML also adds the same **`attention=`** / **`attention_kind=`** banner as **`replayt runs`** / **`replayt report`**). For **`stakeholder`** and **`support`**, HTML output also embeds the same **Stakeholder CLI handoff** block as **`replayt report`** (copy-paste **`replayt inspect`**, report regen, this timeline, **`replayt bundle-export … --report-style …`**, **`replayt resume`** when paused, and JSON **`inspect`** when failed) so a standalone **`timeline.html`** matches the report handoff without opening **`report.html`**. **`bundle-export`** **`timeline.html`** uses the same style as **`--report-style`** and records **`timeline_style`** next to **`report_style`** in **`manifest.json`**.

## `replayt report RUN_ID`

Self-contained HTML report (summary, states, structured outputs, tool calls, token usage, approvals when present), or Markdown via **`--format markdown`** for pasting into tickets, chat, or email (same `--style` rules: **stakeholder** / **support** still omit tool-call and token sections). The header card repeats the same **`attention=`** / **`attention_kind=`** line as **`replayt runs`** text output (from `run_attention_summary`) when a run needs triage, so tickets and **`bundle-export`** HTML match the inventory view. **`--style`:** `default`, **`stakeholder`** (run and approval context first; hides tool-call and token sections), or **`support`** (same hides, plus failure, retry, and parse-failure context first for PM/support handoffs). **`stakeholder`** and **`support`** add a **Stakeholder CLI handoff** section (HTML and Markdown) after the action summary when present: copy-paste lines for **`replayt inspect`**, regenerating this report, offline **`replayt replay`**, **`replayt bundle-export … --report-style …`** (stakeholder tarball: report + timeline + sanitized JSONL + manifest), **`replayt resume TARGET …`** when paused, and **`replayt inspect --output json`** when the run failed. Repeat **`--log-dir`** / **`--log-subdir`** / **`--sqlite`** on each command when you did not use defaults. Paused runs still spell out **`replayt resume`** in the paused callout as well. Approval cards now surface resolver / reason / actor fields from `approval_resolved` plus the applied resume path when present in JSONL. Repeat **`--llm-model MODEL`** (OR semantics) to limit structured outputs, structured parse-failure callouts, and token totals to **`llm_response`** / structured-output lines whose logged model id matches (same rules as **`replayt inspect --llm-model`**). States, approvals, and the state timeline still describe the full run. `--out PATH` writes a file; omit `--out` for stdout.

## `replayt report-diff RUN_A RUN_B`

Side-by-side comparison of two runs (workflow, status, state chain, run metadata / experiment / workflow metadata, failure signals, structured outputs, approvals). ## `replayt validate [TARGET]`

## `replayt runs`

List recent local runs. **`--status completed|failed|paused|unknown`** (repeatable; OR semantics) filters on the best-effort terminal status derived from JSONL the same way as the listing columns (`unknown` when the log has no `run_completed` / `run_paused` yet). ## `replayt doctor`

Check install, env vars, optional YAML extra, default provider connectivity, and best-effort filesystem readiness for the resolved `log_dir` / optional SQLite mirror path. `doctor` also reports soft trust-boundary warnings for risky defaults such as remote plain-HTTP base URLs, embedded credentials in `OPENAI_BASE_URL`, `log_mode=full`, a missing approval-reason policy for paused-run resumes, (on POSIX) a **`log_dir`** that is group- or world-readable/writable, (on POSIX) **`.env`** files under the current working directory or next to the discovered project config file when those files are group- or world-readable/writable (paths and mode bits only; contents are not read), (on POSIX) **inputs JSON** files resolved from **`REPLAYT_INPUTS_FILE`** / project **`inputs_file`** and from **`--inputs-file`** on preflight when those paths exist on disk (**`trust_inputs_file_*`**; skipped for stdin **`--inputs-file -`** / **`REPLAYT_INPUTS_FILE=-`**), (on POSIX) resolved **policy hook** script paths for **`run_hook`**, **`resume_hook`**, **`export_hook`**, **`seal_hook`**, and **`verify_seal_hook`** with the same group/world permission hints (**`trust_policy_hook_script_*`**; best-effort argv parsing; system **`python`** / **`bash`** binaries under standard **`bin`** dirs are skipped), and (on POSIX) when **`--target`** is set, the workflow entry file (`.py` / YAML path or imported module **`__file__`**) with the same group/world permission hints (**`trust_workflow_entry_*`**). When **`forbid_log_mode_full`** or **`REPLAYT_FORBID_LOG_MODE_FULL`** is in effect, **`log_mode_full_forbidden`** fails if the default resolved **`log_mode`** would be **`full`** (so CI **`replayt doctor`** catches a mis-set project default before **`replayt run`** fails). When project config sets **`min_replayt_version`**, `doctor` adds **`project_config_min_replayt_version`**: it is **healthy only** when the installed package meets that floor (or the constraint is unset); invalid constraint strings fail this check too. When **`.replaytrc.toml`** and **`pyproject.toml`** with **`[tool.replayt]`** both exist in the same directory, **`project_config_shadowed_sources`** warns that only the rc file is read (soft; same JSON **`healthy`** rules as unknown keys). JSON output includes **`credential_env`**: a fixed sorted list of common third-party LLM credential variable names (OpenAI-compat keys plus several other popular gateways replayt does not read) each with boolean **`present`** (never values), plus a soft **`credential_env_extra_providers`** check when non-`OPENAI_API_KEY` entries from that list are non-empty in the process environment (replayt's default client does not read them). It also includes **`egress_trust_env`**: the same **`{name, present}`** shape for proxy variables (POSIX lowercase **`http_proxy`** / **`https_proxy`** / **`no_proxy`** / **`all_proxy`** map to the uppercase rows) and common TLS trust-store overrides (**`SSL_CERT_FILE`**, **`REQUESTS_CA_BUNDLE`**, …). It also includes **`ci_artifacts`** plus readiness checks for env-driven CI outputs (`REPLAYT_JUNIT_XML`, `REPLAYT_SUMMARY_JSON`, and `REPLAYT_GITHUB_SUMMARY` / `GITHUB_STEP_SUMMARY`), so a missing parent directory or missing GitHub summary sink can fail preflight before the workflow runs. Optional **`--target TARGET`** plus **`--strict-graph`** / **`--inputs-json`** / **`--inputs-file`** (or default inputs from **`REPLAYT_INPUTS_FILE`** / project **`inputs_file`** when those flags are omitted) / repeatable **`--input key=value`** preflight-loads a workflow and runs the same graph/input validation as `replayt validate` without executing it. **`--format json`** prints `replayt.doctor_report.v1` with a **`healthy`** boolean: exit `0` when healthy, `1` otherwise. **`healthy`** ignores missing **`openai_api_key`**, missing project config, **`project_config_shadowed_sources`**, and these soft trust warnings so CI can validate graphs without secrets; other checks (including **`yaml_extra`**, **`provider_connectivity`** unless `--skip-connectivity`, path-readiness failures, CI artifact readiness failures, **`project_config_min_replayt_version`** when **`min_replayt_version`** is set, and optional `target_validation`) must pass.

## `replayt config`
