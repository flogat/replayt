## 0.4.4 - 2026-03-21

- **SQLite robustness:** failed SQLite writes now roll back their transaction before re-raising, so a single insert failure no longer leaves the store wedged with `cannot start a transaction within a transaction` for the rest of the process; covered by new regressions in [`tests/test_sqlite_store.py`](tests/test_sqlite_store.py).
- **Read-path hardening:** read-only CLI commands now open JSONL stores without creating the log directory as a side effect, so `runs` / `stats` / `inspect` / `report` style commands do not mutate the filesystem just to discover that no logs exist; covered by a new regression in [`tests/test_cli.py`](tests/test_cli.py).
- **Docs:** refresh [`docs/CODE_REVIEW_2026-03-21.md`](docs/CODE_REVIEW_2026-03-21.md) with the current standalone review pass, including the SQLite rollback finding, the read-side log-directory creation finding, mitigation mapping, and targeted verification status.
- **CLI hardening:** project-configured `log_dir` and `sqlite` paths are now resolved relative to the config file that declared them, so runs launched from subdirectories do not silently write logs or SQLite mirrors into the wrong working directory; covered by new CLI regressions in [`tests/test_cli.py`](tests/test_cli.py).
- **CLI correctness:** `replayt run --dry-run --output json` now emits pure JSON instead of prepending the human banner line, so machine-readable output remains parseable in scripting and CI flows; covered by a new regression in [`tests/test_cli.py`](tests/test_cli.py).
- **Docs:** refresh [`docs/CODE_REVIEW_2026-03-21.md`](docs/CODE_REVIEW_2026-03-21.md) with the current standalone review pass, covering config-relative path resolution for `log_dir` / `sqlite`, the `--dry-run --output json` contract bug, mitigations, and targeted verification status.
