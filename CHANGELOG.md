## 0.3.0 — 2026-03-20

User-facing themes below map to commits since git tag **`v0.2.0`**; see [`docs/RELEASE_INVENTORY.md`](docs/RELEASE_INVENTORY.md) for the archived commit ledger covering **`v0.2.0` … `f4a676a`**.

- **Breaking:** the tutorial package on PyPI is now **`replayt_examples`** (was `examples`). Update targets: `replayt run replayt_examples.e01_hello_world:wf` (not `examples.*`).
- **CLI layout:** Typer app split into **`replayt.cli.commands`** (`run`, `inspect`, `export`, `doctor`) plus **`config`**, **`validation`**, **`stores`**, **`targets`**, **`run_support`**, **`ci_artifacts`**, **`display`**, **`constants`**; slim **`main`** registers commands and re-exports.
- **CLI commands:** **`replayt validate`** and **`replayt run --dry-check`** with JSON output (`replayt.validate_report.v1`); **`--inputs-file`** (UTF-8, mutual exclusion with **`--inputs-json`**); **`--strict-graph`** (multi-state workflows with no declared transitions are an error when strict; otherwise **`validate`** / **`dry-check`** emit a warning); **`--experiment-json`** on run (feeds `run_started.experiment` and LLM effective settings); **`runs`** / **`stats`** filters **`--experiment key=value`** and **`--run-meta`**; **`replayt try`** defaults offline (**`--live`** for real LLM); **`replayt ci`** with **`--junit-xml`**, **`--github-summary`** (paths only; **`REPLAYT_JUNIT_XML`** still supported for **`replayt run`**); **`replayt seal`**; **`replayt export-run`** (**`.tar.gz`** + sanitization + bundled log JSON Schema); **`replayt bundle-export`**; **`report-diff`** HTML; richer stakeholder **`replayt report`** (approvals, timestamps); **`replayt doctor --format json`** (`replayt.doctor_report.v1`, **`healthy`** + exit); **`replayt init --ci github`**; **`replayt resume`** **`--reason`**, **`--actor-json`**, **`--resolver`**, optional **`resume_hook`** / **`REPLAYT_RESUME_HOOK`** with default **120s** subprocess timeout (**`resume_hook_timeout`** / **`REPLAYT_RESUME_HOOK_TIMEOUT`**; ≤ 0 = unlimited).
- **Logs & metadata:** **`REPLAYT_LOG_DIR`**, **`--log-subdir`**, **`--metadata-json`**; validate that the workflow’s initial state is a declared step before run.
- **Runner & workflow:** optional **`Runner(..., before_step=..., after_step=...)`**; optional **`Workflow(..., llm_defaults=...)`** and **`meta["llm_defaults"]`** merged into **`LLMBridge`** defaults (omitted from logged **`workflow_meta`**); YAML infers **`note_transition`** edges from **`next`** / **`branch`** / **`approval`** for graph validation; **`run_started`** carries workflow meta; LLM **`with_settings(experiment=...)`**; **`ReplaytError`** (e.g. **`RunFailed`**) from a step does not consume retry attempts; approval resume pairs **`approval_resolved`** with **`approval_requested`** in **FIFO** order per id; per-run approval state reset; **`resolve_approval_on_store`** resolves the **oldest** pending request for that id; store records **`approval_resolved`** **`approval_id`** as string.
- **Persistence:** **`make_store`** used from timeout/interrupt paths so **`run_interrupted`** reaches SQLite mirrors when **`--sqlite`** is set; **`MultiStore.close()`** closes SQLite mirrors; **`strict_mirror`** defaults **true** whenever **`--sqlite`** (or config **`sqlite`**) is used unless **`strict_mirror`** is set explicitly; **`SQLiteStore`**: WAL mode, **`__enter__`/`__exit__`**, corrupt **`payload_json`** surfaced as **`RuntimeError`** with **`run_id`** and **`seq`**; JSONL tail fallback when the last line is unreadable; lock failures raise **`LogLockError`**; lock failures logged on POSIX and Windows; Windows empty-file lock sentinel; **`replayt seal`** validates **`run_id`** like stores and refuses JSONL paths outside the resolved log directory.
- **LLM & tools:** **`llm_coercion`** for temperature / max_tokens / timeout (YAML + **`LLMBridge`**); **`OpenAICompatClient`** reads chat completions with **streaming** and enforces **`LLMSettings.max_response_bytes`** (env **`REPLAYT_LLM_MAX_RESPONSE_BYTES`**); **`LLMSettings.from_env`** validates **`REPLAYT_LLM_MAX_RESPONSE_BYTES`** and **`REPLAYT_LLM_MAX_SCHEMA_CHARS`**; **`LLMBridge.parse`** rejects JSON Schemas over **`max_schema_json_chars`** (default 250k) and responses over **`max_parse_response_chars`** (default 4M); clearer error when the model returns no JSON object; HTTP retries treat **500** as retryable; **`_extract_json_object`** caps **`{`** scan work and, after dropping dict spans nested inside a larger dict, takes the **last** remaining span; **`coerce_max_tokens_for_api`** rejects booleans with **`TypeError`** (same idea as temperature); **`ToolRegistry`** resolves **`get_type_hints`** with the defining module’s globals, with **`Any`** fallback + warning on unresolved forward refs; YAML LLM prompts interpolate only **named placeholders** (valid identifiers); **`collect_report_context`** and **`build_run_report_html`** both use **`aggregate_run_report_data`** (single pass over events); float **`Retry-After`** parsing; **`DryRunLLMClient`** nested **`$ref` / allOf / oneOf`** minimal schema fill.
- **Fixes (after `v0.2.0` tag):** CLI tool/report/diff paths use **`name`** / **`arguments`** to match **`tools.py`** emissions; YAML init template uses **`next`** (not **`transition`**); **`Runner`** context manager closes **`httpx`**; lazy **`httpx.Client`** in **`OpenAICompatClient`** for dry/mock clients; deep-copy context in snapshots for pause/resume; **`diff`** gains **`--sqlite`**; **`MultiStore.append`** catches mirror failures like **`append_event`**; clip-path fix for demo SVGs so replay command text stays visible.
- **CI & tests:** PR gate requiring **`CHANGELOG.md`** when protected paths change; test that tutorial **`MODULE:wf`** targets from the examples README stay importable.
- **Documentation:** **[`docs/PRODUCTION.md`](docs/PRODUCTION.md)**, **[`docs/RECIPES.md`](docs/RECIPES.md)**, **[`docs/INSTALL.md`](docs/INSTALL.md)**, **[`docs/CLI.md`](docs/CLI.md)**, **[`docs/CONFIG.md`](docs/CONFIG.md)**, **[`docs/COMPARISON.md`](docs/COMPARISON.md)**, **[`docs/EXAMPLES_PATTERNS.md`](docs/EXAMPLES_PATTERNS.md)**; **[`docs/QUICKSTART.md`](docs/QUICKSTART.md)** and **[`docs/SCOPE.md`](docs/SCOPE.md)**; README slimming and onboarding (recipes extracted, comparison / install / report); quickstart “replay vs regenerate” diagram + PII note; PyPI **Beta** classifier; four animated SVG demos (core loop, approval gate, failure debug, framework comparison); README and tutorial README composition notes (streaming, planners, agent-framework boundaries); plainer wording in README, **`docs/README`**, **`docs/COMPARISON`**, tutorial README; **`MultiStore` / `strict_mirror`**, trusted **`resume_hook`**, and related notes in **`CONFIG.md`**, **`PRODUCTION.md`**, and code docstrings; maintainability: approval map rename in resume path; resume snapshot uses **`is not None`** so an empty string state name is respected; sanitized Mermaid node IDs in notebook helper.

## 0.2.0 — 2026-03-20

- Documentation: add [`docs/QUICKSTART.md`](docs/QUICKSTART.md) and [`docs/SCOPE.md`](docs/SCOPE.md); tighten README above-the-fold (comparison table, `pip install replayt`, `replayt report`, links); trim [`docs/DEMO.md`](docs/DEMO.md) to checklist; fix examples README link to [`docs/STYLE.md`](docs/STYLE.md); PyPI classifier **Beta**.
- HTML-escape all user-facing values in `RunResult._repr_html_` to prevent XSS in notebooks.
- Harden YAML workflow prompt interpolation against attribute-access injection (`format_map` replaced with safe formatter).
- Reuse `httpx.Client` across LLM calls for connection pooling and lower latency.
- Add opt-in HTTP retry via `LLMSettings(http_retries=N)` for transient errors (429, 502, 503, 504) with exponential backoff. Default is `0` (no retries) to stay explicit per design principle #2.
- Fix `DryRunLLMClient._minimal_json_from_schema` dead-code condition that included all fields regardless of `required`.
- Fix `tool-using` init template to match actual `ToolRegistry.register()` API.
- `MultiStore` mirror writes are now best-effort; failures are logged instead of crashing the run.
